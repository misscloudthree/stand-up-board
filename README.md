# 站會看板 Standup Board

團隊站會用的追蹤工具，網址：**https://misscloudthree.github.io/stand-up-board/**

沒有自己的後端伺服器，所有資料都寫在**這個 repo 的 GitHub Issues** 裡，所以大家看到的是同一份資料。

## 功能

- **Backlog 追蹤**：站會議題看板，可設類型/負責人/deadline/狀態，卡片可留言討論。工具列可依狀態、類型、負責人篩選，依 deadline 排序。
- **每日測試追蹤**：每個模組每天輸入追蹤數字，自動加總 Summary；下方有依週/依月的長條圖看趨勢。
- **交付測試追蹤**：依模組總覽交付/FT/SIT 測試狀態、測試完成度、風險項目，可依交測客戶日期月份分群看已交付/未交付狀況，支援 Excel 範本下載與批次上傳更新。
- **看板設定項目**：集中管理「FSD 模組」與「每日測試追蹤項目」，頁面上有寫每項設定會影響哪些頁籤。

## 加入使用（新成員必看）

寫入資料需要你自己的 GitHub Token，而 Token 要能運作，你必須先被加入這個 repo。**順序不能顛倒**，沒完成第 1-2 步，第 3 步會選不到這個 repo。

### 1. 請站長把你加為 collaborator
請站長（repo 擁有者）做：repo 頁面 → **Settings** → **Collaborators and teams** → **Add people** → 輸入你的 GitHub 帳號 → 權限選 **Write**。

### 2. 接受邀請
你會收到 GitHub 通知信，或直接開 [invitations 頁面](https://github.com/misscloudthree/stand-up-board/invitations) 按 **Accept invitation**。沒接受邀請前，下一步的 Token 設定會看不到這個 repo。

### 3. 建立你自己的 Personal Access Token
1. GitHub 右上角頭像 → **Settings** → **Developer settings** → **Personal access tokens** → **Fine-grained tokens** → **Generate new token**
2. **Repository access** 選 **Only select repositories** → 勾選 `stand-up-board`
3. **Permissions → Repository permissions → Issues** 設為 **Read and write**
4. 產生後複製 token（`github_pat_...` 開頭），**只會顯示一次，要先存好**

### 4. 到網站貼上 Token 跟顯示名稱
打開網站 → 右上角 **⚙ 設定** → 貼上 Token、填上你的顯示名稱 → 儲存。

> ⚠️ **顯示名稱不是身分綁定**，只是存在你瀏覽器裡的一段文字，用來當「提出人/回報人」預設值。換裝置、清過瀏覽器資料、或重辦 Token 後，**顯示名稱要打得跟之前一模一樣**（大小寫、全形半形都算），不然系統會當成不同人，篩選負責人時會對不起來。建議團隊先講好每個人的名稱要怎麼打。

Token 只存在你自己瀏覽器的 localStorage，不會給其他人看到；每人要各自建立自己的 Token，不要共用。

## 使用方式

**Backlog 追蹤**
- 「＋ 新增 Backlog」建立議題；點卡片可改內容/類型/負責人/deadline/狀態，下面可留言記錄進度
- deadline 兩天內或已逾期會自動標色
- 工具列：狀態勾選（預設只顯示待處理/進行中）、類型篩選、負責人篩選、deadline 排序

**每日測試追蹤**
- 先到「看板設定項目」新增模組、設定要追蹤的欄位
- 選日期、每個模組填數字、按「儲存」；下方 Summary 自動加總
- 別人更新資料後按「重新整理」才會看到（非即時同步）
- 「趨勢圖表」可切依週（近 8 週）／依月（近 6 個月），畫成長條圖
- ⚠️ 同一個模組、同一天的資料是單一筆記錄，**不是逐欄合併**：如果兩人先後對同一個模組填同一天的數字並儲存，後存的人會整列蓋掉先存的人。建議約定每個模組每天固定一人回報，或填寫前先按「重新整理」拿最新資料

**交付測試追蹤**
- 「下載 Excel 範本」→ 填好資料 → 「上傳 Excel 更新」批次匯入
- 上傳會先檢查每一列必填欄位，有錯誤會列出「第幾列/哪個欄位」，**全部通過才會匯入**，不會匯入一半；日期欄空白或格式無法辨識一律視為未設定，**不會擋上傳**
- 風險判斷：「測試設定」可調整全域「基本測試天數」（每個功能項目也可個別覆寫）；系統用「交測客戶日期」－「預計開發交付時程」算出可測試天數，小於基本測試天數就標示「測試時程不足」
- 模組卡片顯示交付狀態、FT/SIT 測試狀態分布、測試完成度（皆已測通／只測通FT尚待SIT／尚未完成），可展開看每個功能項目明細
- 上方「依交測客戶日期分群」依月份彙總已交付（超過交測客戶日期且 FT/SIT 都測通或不需測試）／未交付數量

**看板設定項目**
- 「FSD 模組」跟「每日測試追蹤項目」在這裡統一管理，改了會影響其他頁籤的顯示

## 已知限制

- 非即時同步，需要手動按「重新整理」；「每日測試追蹤」同模組同一天的資料後存會整列蓋掉先存的（見上方「每日測試追蹤」用法）
- 一般清單（Backlog、模組、追蹤項目）一次抓 100 筆，超過的話舊項目可能看不到；「每日測試追蹤」趨勢圖表的歷史資料有做分頁，不受此限
- Token 跟顯示名稱都只存在瀏覽器裡：共用電腦用完建議清除，或用權限最小的 fine-grained token；換裝置需要手動重新輸入且要打一模一樣（見上方「加入使用」第 4 步）
- 「交付測試追蹤」的測試情境資料是每個模組合併成一個 GitHub issue（避免資料量大時超過 API 速率限制），所以單一測試情境沒有自己的 issue、無法個別留言
- Excel 功能透過外部 CDN 載入函式庫，離線環境無法使用
- 想看原始資料：到 repo 的 **Issues** 分頁即可
